You are a Senior Resource Manager and Delivery Lead with expertise in 
planning and staffing digital product initiatives.

Using the attached BRD, FSD, TDD, and Project Plan / WBS as your 
source of truth, generate a comprehensive Resource Plan for the 
App Adoption (DIY Servicing Feature).

The document must include:

1. **Resource Planning Summary**
   - Total headcount required
   - Total estimated person-days
   - Planned project duration
   - Sourcing strategy: Internal vs External vs Vendor

2. **Role-wise Resource Breakdown**
   For each role, specify:
   - Role title and seniority level
   - Number of resources
   - Responsibilities (mapped to WBS workstreams)
   - Required skills and tools
   - Engagement type: Full-time / Part-time / Advisory
   - Phase(s) of involvement

   Cover these roles at minimum:
   * Project Manager
   * Business Analyst
   * UX Designer / UI Designer
   * Frontend Developer (Mobile + Web)
   * Backend Developer
   * API / Integration Developer
   * QA Engineer (Manual + Automation)
   * DevOps / Infrastructure Engineer
   * Solution Architect / Tech Lead
   * Change Management / Training Lead
   * Product Owner
   * Scrum Master

3. **Resource Allocation Matrix (RAM)**
   - Map each resource role to project phases
   - Show allocation % per phase
   - Identify ramp-up and ramp-down timelines
   - Format: Role | Phase 1 | Phase 2 | ... | Phase N | Total Days

4. **RACI Matrix**
   - For each major deliverable and milestone
   - Roles: Responsible, Accountable, Consulted, Informed
   - Cover: BRD sign-off, FSD sign-off, UX approval, Dev completion,
     QA sign-off, UAT, Go-Live, Hypercare exit

5. **Skill Gap Analysis**
   - Identify skills not available internally
   - Recommend: Hire / Contract / Train / Vendor
   - Flag critical skill gaps as [GAP - ACTION NEEDED]

6. **Onboarding & Knowledge Transfer Plan**
   - Onboarding timeline for new resources
   - Knowledge transfer checkpoints between phases
   - Documentation handoff requirements

7. **Resource Cost Estimate**
   - Derive from BRD development cost (₹80,00,000 budget)
   - Role-wise cost breakup: Daily rate × Person-days
   - Distinguish one-time vs ongoing resource costs
   - Flag where rates are assumed as [ASSUMED RATE]

8. **Backup & Contingency Plan**
   - Key person dependencies and single points of failure
   - Backup resource strategy for critical roles
   - Bench or shadow resource recommendations

9. **Resource Risks & Mitigations**
   - Attrition risk
   - Skill unavailability
   - Cross-team dependency conflicts
   - Severity: 🔴 Critical | 🟡 Moderate | 🟢 Minor

10. **Governance & Reporting**
    - Resource utilisation tracking cadence
    - Escalation path for resource conflicts
    - Tools recommended: JIRA / MS Project / Smartsheet

Output Format:
- Structured document with clear section headers
- Use tables for RAM, RACI, cost breakup, and skill gap
- All cost figures in INR (₹)
- Flag decisions needing approval as [DECISION NEEDED]
- Flag assumptions as [ASSUMED]
