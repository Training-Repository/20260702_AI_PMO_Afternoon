You are a Senior Project Manager with expertise in digital product 
delivery using Agile-Waterfall hybrid methodologies.

Using the attached BRD, FSD, and TDD as your source of truth, generate 
a comprehensive Project Plan and Work Breakdown Structure (WBS) for the 
App Adoption (DIY Servicing Feature).

The document must include:

1. **Project Summary**
   - Objective, scope boundaries, and delivery approach
   - Reference to BRD business objectives and success metrics

2. **Project Phases & Milestones**
   - Define phases: Initiation → Planning → Design → Development 
     → Testing → UAT → Go-Live → Hypercare
   - Key milestone dates and deliverables for each phase
   - Flag dependencies between phases

3. **Work Breakdown Structure (WBS)**
   - Break down work to task level across all phases
   - Structure: Phase → Workstream → Activity → Task
   - Cover these workstreams:
     * Product & Business Analysis
     * UX / UI Design
     * Frontend Development (Mobile + Web)
     * Backend / API Development
     * Integration & Middleware
     * QA & Testing
     * DevOps & Infrastructure
     * Training & Change Management
     * Project Management & Governance

4. **Effort & Duration Estimates**
   - Estimated effort (person-days) per task
   - Duration in calendar days
   - Assumptions behind estimates

5. **Resource Plan**
   - Roles required, number of resources, allocation %
   - Derive team size from BRD cost estimates where possible

6. **Sprint Plan (Agile Layer)**
   - Suggested sprint breakdown (2-week sprints)
   - Map FR-01 to FR-05 features to sprints
   - Include buffer sprints for bug fixing and UAT feedback

7. **Critical Path**
   - Identify the sequence of tasks that determine the 
     earliest project completion
   - Highlight top 3 critical path risks

8. **RAID Log (Initial)**
   - Risks, Assumptions, Issues, Dependencies
   - Derive from BRD risks section and open questions

9. **Communication & Governance Plan**
   - Meeting cadence, reporting frequency, escalation matrix

10. **Project Closure Criteria**
    - Definition of Done aligned to BRD success metrics
    - Sign-off requirements

Output Format:
- Structured document with clear section headers
- Use tables for WBS, resource plan, and RAID log
- Use numbered WBS codes (1.0, 1.1, 1.1.1 format)
- Flag items needing stakeholder input as [INPUT NEEDED]
- Flag assumptions made due to missing info as [ASSUMED]
